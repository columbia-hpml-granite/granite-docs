# Adding Images to Weekly Updates

## Quick Reference

### Image Locations
- **Week 1:** `/static/img/weekly/week1/`
- **Week 2:** `/static/img/weekly/week2/`
- **Week 3:** `/static/img/weekly/week3/`
- etc.

### Markdown Syntax
```markdown
![Description](/img/weekly/week2/chart-name.png)
```

## Example for Week 2

If you have these images for Week 2:
- `encoder-decoder-breakdown.png`
- `dtype-comparison.png`
- `token-length-comparison.png`

Place them in: `/static/img/weekly/week2/`

Then reference in `week2.md`:

```markdown
### Encoder-Decoder Breakdown Results

![Encoder-Decoder Latency](/img/weekly/week2/encoder-decoder-breakdown.png)
*Figure 1: Encoder vs Decoder latency across different audio lengths*

### DType Comparison

![bf16 vs fp16 Performance](/img/weekly/week2/dtype-comparison.png)
*Figure 2: Performance comparison between bf16 and fp16 precision modes*
```

## Tips

1. **File names:** lowercase-with-hyphens.png
2. **Formats:** PNG for charts, SVG for diagrams
3. **Always include alt text** for accessibility
4. **Add captions** using italic text below images
